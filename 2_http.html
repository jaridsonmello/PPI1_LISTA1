Protocolo HTTP

A linguagem utilizada na comunicação entre cliente e servidor é o que chamamos de protocolo, no caso da web o HTTP. O HTTP é um protocolo de comunicação que permite a transferência de informações na web. Ele é responsável por estabelecer a comunicação entre o cliente e o servidor, permitindo que as informações sejam transmitidas de forma segura e eficiente.

O HTTP é um protocolo de comunicação baseado em texto, o que significa que as mensagens são enviadas em formato de texto, facilitando a leitura e a interpretação dos dados, é um protocolo de comunicação sem estado, o que significa que ele não mantém informações sobre as requisições anteriores, cada requisição é tratada de forma independente.

Ele é um protocolo de comunicação baseado em requisição-resposta, o cliente envia uma requisição ao servidor e o servidor responde com uma resposta. O HTTP é um protocolo de comunicação baseado em conexão, o cliente estabelece uma conexão com o servidor para enviar e receber informações.

O HTTP é um protocolo de comunicação baseado em métodos, o cliente pode enviar diferentes tipos de requisições ao servidor, como GET, POST, PUT, DELETE, entre outros.

É um protocolo de comunicação baseado em códigos de status, o servidor responde com um código de status que indica o resultado da requisição, como 200 (OK), 404 (Not Found), 500 (Internal Server Error), entre outros. Também é baseado em cabeçalhos, o cliente e o servidor podem enviar informações adicionais nos cabeçalhos das mensagens, como cookies, autenticação, cache, entre outros. O cliente pode acessar recursos na web através de URLs, como http://www.exemplo.com/pagina.html.

O HTTP é um protocolo de comunicação baseado em versões, o HTTP possui diferentes versões, como HTTP/1.0, HTTP/1.1, HTTP/2, HTTP/3, cada versão possui melhorias e novas funcionalidades. O HTTP pode ser utilizado de forma segura através do HTTPS, que utiliza criptografia para proteger as informações transmitidas na web.

Request

Quando um cliente faz uma solicitação a um servidor é enviada uma mensagem HTTP pela portas 80 ou 443. O servidor então processa a mensagem e gera um texto de retorno, o response HTTP

Response

Quando a resposta da requião chega ao cliente ela é processada pela aplicação cliente, que pode exibi-la ou interpreta-la.
